# Step 2: 3選の構成案

**テーマ:** Claude Code — 開発者に役立つ便利機能3選

---

## 構成

### 🎬 選1: MCP サーバー連携 — 「ターミナルから出ない開発体験」
- **選定理由:** 開発者なら誰でも感じる「ツール切り替えのストレス」に直結。わかりやすく、インパクト大。
- **ポイント:** GitHub PRの作成、データベースクエリ、Slack通知——すべてターミナルの中で完結する。
- **順序の意図:** 最もわかりやすい機能から入り、視聴者の興味を掴む。

### 🎬 選2: カスタムサブエージェント — 「AIが分身して並列で働く」
- **選定理由:** 「AIが複数同時に動く」というビジュアルイメージが強く、短尺でも伝わる。技術的にも一歩深い。
- **ポイント:** 大きなタスクを分割→複数のAIエージェントが並列で作業→最後にマージ。
- **順序の意図:** 選1で掴んだ興味を「さらにすごい」方向に引き上げる。

### 🎬 選3: Skills（スキル）— 「この動画も、実はClaude Codeが作ってます」
- **選定理由:** メタ要素の種明かしに最適。「スキル」という仕組みで動画生成パイプラインまで自動化できることを示す。
- **ポイント:** カスタムワークフローを定義して `/コマンド` 一発で実行。台本作成→画像生成→動画生成まで自動化。
- **順序の意図:** 最後に「実はこの動画自体がその機能で作られている」と種明かしすることで、視聴者に強い印象を残す。

---

## 構成の流れ

```
選1（わかりやすさ）→ 選2（技術的深さ）→ 選3（メタ種明かし・サプライズ）
```

基礎→応用→驚き の3段構成で、30秒間飽きさせない設計。
