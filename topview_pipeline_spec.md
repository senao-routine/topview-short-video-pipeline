# Topview AI × Claude Code 全自動ショート動画生成パイプライン 要件定義書

**バージョン:** 1.0
**作成日:** 2026年4月11日
**対象:** Claude Code(Topview Skill インストール済み環境)
**目的:** ユーザーが渡したテーマから、Pixar風3Dキャラクターが解説する30秒ショート動画を半自動で生成するパイプラインの構築

---

## 1. 目的とゴール

### 1.1 最終成果物

ユーザーから渡された「動画テーマ」を起点として、以下を自動生成し、最終的に**9:16縦型の30秒ショート動画ファイル**を出力する。

- 指定テーマに関する情報収集結果
- 「〇〇できること3選」形式の構成案
- 10秒×3シーン = 合計30秒のセリフ台本
- Pixar風3Dキャラクター画像(Nano Banana 2経由で生成)
- リップシンク付き動画(Topview AIアバター機能経由で生成)

### 1.2 ゴールの定義

**「ユーザーがブラウザを一切開かずに、Claude Codeのターミナル上での対話だけで完成動画を受け取れる状態」** を達成することがゴール。

---

## 2. 前提条件

以下の環境が整っていることを前提とする:

- [x] Claude Code がインストール・認証済み
- [x] Topview Skill が Claude Code に インストール済み(`https://github.com/topviewai/skill.git`)
- [x] Topview AI アカウントが認証済み、API連携有効
- [x] Topview AI の AIアバター機能が利用可能なプラン契約中
- [x] Nano Banana 2(画像生成モデル)に Topview Skill 経由でアクセス可能
- [x] 作業ディレクトリに読み書き権限あり

---

## 3. 入力仕様

### 3.1 ユーザーからの入力形式

ユーザーは以下の形式で「動画テーマ」を1段落で渡す:

```
【テーマ】[トピック名]
【3選の観点】[どんな切り口で3選を選ぶか]
【ターゲット視聴者】[想定視聴者層]
【トーン】[親しみやすい/専門的/カジュアル 等]
【その他要望】[任意]
```

### 3.2 入力例(本番収録で使用予定)

```
【テーマ】Claude Code
【3選の観点】開発者に役立つ便利機能
【ターゲット視聴者】AI・プログラミングに関心のある開発者
【トーン】親しみやすく、でも技術的な正確さは保つ
【その他要望】最後の1選にメタ要素(「この動画もClaude Codeで作ってます」的な種明かし)を入れたい
```

## 4. 処理フロー

本パイプラインは**2箇所のユーザー確認ポイント**を含む半自動フローとして設計する。

### フロー全体図

```
[入力受付]
   ↓
[Step 1] 情報収集
   ↓
[Step 2] 3選選定と構成案作成
   ↓
[Step 3] 10秒×3のセリフ台本生成
   ↓
【🛑 停止ポイント1】台本確定
   ├─ 承認 → 次へ
   └─ 修正指示 → Step 3 に戻る(ループ)
   ↓
[Step 4] キャラクター画像生成(Nano Banana 2)
   ↓
【🛑 停止ポイント2】画像確定
   ├─ 承認 → 次へ
   └─ 修正指示 → Step 4 に戻る(ループ、または同プロンプトで再生成)
   ↓
[Step 5] Topview AIアバター機能で動画生成
   ↓
[Step 6] 完成動画の出力と提示
```

### 4.1 Step 1:情報収集

**処理内容:**
- ユーザーから受け取った「テーマ」に関する最新情報を Web 検索等で収集する
- 3選の候補となりうる機能・特徴・Tips を 5〜7 個リストアップする
- 各候補について、なぜ視聴者に刺さるかの根拠を簡潔にメモする

**出力:**ターミナルに情報収集結果のサマリーを表示(ユーザー確認は不要、参考情報として表示)

**制約:**
- 信頼性の高いソース(公式サイト、公式ドキュメント、著名な技術ブログ等)を優先する
- 古い情報(1年以上前)は参考程度に留める

---

### 4.2 Step 2:3選の選定と構成案作成

**処理内容:**
- Step 1 でリストアップした候補から、以下の基準で3つを選ぶ:
 - 視聴者(ターゲット層)に最も刺さる
 - 30秒という短尺で伝えやすい
 - 3つ並べた時に飽きないバリエーションがある
- 選んだ3つを、視聴者の興味を引く順序で並べる(強い→弱い or 基礎→応用 等)
- ユーザーの要望に「メタ要素」等の特別指定がある場合は、該当する選に組み込む

**出力:**ターミナルに3選の構成案を表示

---

### 4.3 Step 3:10秒×3のセリフ台本生成

**処理内容:**
- 各選について、10秒程度で読み切れる日本語のセリフを1つずつ作成する
- 目安:1セリフあたり 70〜90文字(話速に依存)
- トーンはユーザー指定に従う(デフォルトは親しみやすい女性キャラの一人語り)
- 視聴者(カメラ)に語りかける形式にする

**台本フォーマット:**

```
━━━━━━━━━━━━━━━━━━━━━━
🎬 シーン1(0:00-0:10)【選1のタイトル】
━━━━━━━━━━━━━━━━━━━━━━
セリフ:「...(70〜90文字)」
(文字数:XX文字)

━━━━━━━━━━━━━━━━━━━━━━
🎬 シーン2(0:10-0:20)【選2のタイトル】
━━━━━━━━━━━━━━━━━━━━━━
セリフ:「...(70〜90文字)」
(文字数:XX文字)

━━━━━━━━━━━━━━━━━━━━━━
🎬 シーン3(0:20-0:30)【選3のタイトル】
━━━━━━━━━━━━━━━━━━━━━━
セリフ:「...(70〜90文字)」
(文字数:XX文字)

📊 合計:XXX文字(目安:210〜270文字)
```

---

### 4.4 🛑 停止ポイント1:台本確定

**処理内容:**
- Step 3 で生成した台本をユーザーに提示する
- ユーザーの明示的な承認または修正指示を**必ず待つ**
- 自動で次のステップに進んではならない

**ユーザーが取りうるアクション:**

| アクション | キーワード例 | システムの動作 |
|---|---|---|
| 承認 | 「OK」「これで進めて」「承認」 | Step 4 へ進む |
| 全体修正 | 「もっと親しみやすく」「テーマを変えたい」 | Step 3 に戻り、指示を反映して再生成 |
| 部分修正 | 「シーン2のセリフをこう変えて」「文字数を減らして」 | 該当箇所のみ修正し、再提示 |
| 中断 | 「中止」「やめる」 | パイプライン全体を中断 |

**重要な挙動:**
- 修正指示を受けた場合、**最新の台本のみを返す**(過去バージョンを混在させない)
- 修正ループは何回でも許容する(上限なし)
- ユーザーが明示的に承認するまで次のステップには進まない

---

### 4.5 Step 4:キャラクター画像生成(Nano Banana 2)

**処理内容:**
- Topview Skill 経由で Nano Banana 2 モデルを呼び出す
- 以下のプロンプトを使用してキャラクター画像を生成する
- アスペクト比は 9:16(縦型)で出力

**使用するプロンプト(確定版):**

```
A Pixar-style 3D animated character for a vertical short video (9:16 
aspect ratio). 

STYLE: Pixar 3D animation aesthetic, similar to Monsters Inc. and 
Inside Out. Smooth polished surfaces, colorful, friendly yet slightly 
cool. Not overly cute, with a hint of coolness.

CHARACTER: A single cute female character, looks like early 20s, 
approachable and intelligent, engineer-type girl.

APPEARANCE:
- Short bob or medium-length dark brown hair
- Wearing large round-frame or black-rimmed glasses
- Big expressive eyes with the lively Pixar-character look
- Casual engineer-style outfit: graphic T-shirt under a relaxed 
  hoodie or cardigan, denim or chino pants
- Overall muted palette (white, gray, navy, beige) with one pop 
  accent color (e.g., yellow sneakers)
- Energetic, bright, talkative personality expression

BACKGROUND: Clean modern creator's workspace. Large desk, dual 
monitors, keyboard, indoor plants, stylish posters and lights on 
the wall. A bit futuristic and tech-vibe workspace. Warm lighting 
with cozy atmosphere.

COMPOSITION: Medium shot from chest up, character facing camera, 
ready to speak. Vertical 9:16 format. Character positioned in 
upper-center of frame to allow space for subtitles at bottom.

QUALITY: Ultra high quality, Pixar film production level, 4K, 
detailed lighting, character clearly visible and in sharp focus.
```

**出力:**生成された画像をローカルに保存し、ターミナルにプレビューまたはパスを表示

---

### 4.6 🛑 停止ポイント2:画像確定

**処理内容:**
- Step 4 で生成した画像をユーザーに提示する
- ユーザーの明示的な承認または修正指示を**必ず待つ**

**ユーザーが取りうるアクション:**

| アクション | システムの動作 |
|---|---|
| 承認 | Step 5 へ進む |
| 同プロンプトで再生成 | 同じプロンプトで Step 4 を再実行 |
| プロンプト微調整 | 指示を反映してプロンプトを修正し、Step 4 を再実行 |
| 中断 | パイプライン全体を中断 |

**重要な挙動:**
- ユーザーが明示的に承認するまで動画生成には進まない
- 再生成は何回でも許容する

---

### 4.7 Step 5:Topview AIアバター機能で動画生成

**処理内容:**
- 承認済みのキャラクター画像をベース画像として Topview の AIアバター機能に渡す
- 確定済みのセリフ(シーン1〜3)をテキスト入力として渡す
- リップシンク付きで各シーンを生成する
- 3シーンを1本の動画として結合する
- 9:16 縦型、合計30秒の動画として出力する

**Topview Skill 経由で実行するパラメータ:**

```
- base_image: [Step 4 で承認された画像パス]
- script_scene1: [確定済みシーン1セリフ]
- script_scene2: [確定済みシーン2セリフ]
- script_scene3: [確定済みシーン3セリフ]
- aspect_ratio: 9:16
- total_duration: 30s
- lip_sync: enabled
- voice: Japanese female, friendly tone
```

**注意:**
- 実際のTopview Skill API仕様に合わせてパラメータ名を調整すること
- リップシンク精度が不安定な場合は、1シーンずつ生成して後から結合する代替フローを試みる

---

### 4.8 Step 6:完成動画の出力と提示

**処理内容:**
- 生成された動画ファイルをローカルディレクトリに保存
- ファイル名は `topview_short_YYYYMMDD_HHMMSS.mp4` 形式
- 保存先パスをターミナルに表示
- 簡単なサマリー(テーマ、3選、総尺、ファイルサイズ)を表示

## 5. エラーハンドリング

### 5.1 各ステップで想定されるエラー

| ステップ | 想定エラー | 対応 |
|---|---|---|
| Step 1 | 情報収集失敗(ネット接続等) | エラー報告、ユーザーに手動情報提供を促す |
| Step 3 | セリフ文字数が目安を超過 | 自動で短縮を試み、結果を提示 |
| Step 4 | 画像生成失敗(API エラー) | エラー報告、3回まで自動リトライ、それでも失敗なら中断 |
| Step 4 | キャラの見た目が指示と異なる | 停止ポイント2でユーザー判断 |
| Step 5 | リップシンク不自然 | 1シーンずつ生成する代替フローに切り替え |
| Step 5 | 動画生成失敗 | エラー報告、2回まで自動リトライ |

### 5.2 中断と再開

- ユーザーが任意のタイミングで「中止」と入力した場合、現在の状態を保存してパイプラインを終了する
- 保存した状態から再開できるよう、各ステップの成果物はローカルに記録する

## 7. 制約事項

以下は絶対に守ること:

1. **著作権の遵守:**
 - 既存の特定アニメ・映画キャラクターを模倣しない
 - 実在人物の顔を参照しない
 - 商標・ロゴを画面に含めない

2. **ユーザー確認の尊重:**
 - 停止ポイント1・2で必ず停止する
 - 承認なしに次のステップに進まない
 - 修正ループを省略しない

3. **透明性:**
 - 各ステップで何をしているかターミナルに逐次表示する
 - エラーは隠さず報告する
 - 自動リトライの回数も明示する

4. **コスト配慮:**
 - Topview API の呼び出し回数を必要最小限にする
 - 不要な生成は行わない
